# Automated tests

Automated tests (pytest) to validate that the multi-perspective
conformance checking algorithm correctly identifies violations.

## Structure

- `conftest.py` — adds `Algorithm/` to `sys.path`, defines input file
  paths, and creates session fixtures that run the algorithm only once per
  input combination.
- `test_with_violations.py` — uses files from `Algorithm/DevelopmentFiles/`
  (with violations already mapped in `violationTestMapping.txt`) and
  verifies that each expected violation is detected and classified in the
  correct category.
- `test_without_violations.py` — uses synthetic error-free logs from
  `ModelosLogsTeste/` (`SyntheticProcessLogTenCasesNoErrors.xes` and
  `SyntheticDataAccessLogTenCasesNoErrorsAcesso.xes`) and checks that the
  report contains no violations.

## How to run

From the repository root, with the algorithm dependencies installed
(see `Algorithm/requirements.txt`):

```powershell
pip install pytest
pip install -r Algorithm/requirements.txt
pytest Test
```

The report files generated by the algorithm as a side effect
(`report*Acesso.txt`) are written to a temporary directory created by
pytest, so the workspace remains clean.

## Adding new test cases

To test with new input files, add a new fixture in `conftest.py`
reusing `_run_algorithm` and write a new `test_*.py` module with
assertions on the returned `report`.
